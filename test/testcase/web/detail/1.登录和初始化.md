# 测试用例：登录和初始化

## 前置条件
- 后端 local 环境已启动
- Web 前端 http://localhost:5173 已启动

## 测试步骤

### TC-001: 用户登录
1. 打开 http://localhost:5173
2. 点击"我的"->"设置"
3. 点击登录按钮
4. 输入账号：admin，密码：admin123
5. 点击确认登录

**预期结果：**
- 登录成功，显示用户信息
- 可以正常访问其他功能

**验收标准：**
- [ ] 登录接口返回 200
- [ ] 本地存储包含用户 token
- [ ] 页面显示已登录状态

---

### TC-002: 进入章节调试初始化流程验证
1. 登录后，点击"我的"->"草稿箱"
2. 点击第一个草稿故事
3. 点击第一章节
4. 点击"调试"

**预期结果：**
- 网络请求顺序正确：
  - 首先调用 `/initDebug` 或等效的初始化接口
  - 然后加载章节内容
  - 然后调用 introduction（如有开场白）
  - 然后调用编排（orchestration）
  - 最后显示台词

**验收标准：**
- [ ] 没有重复的 saveWorld 请求
- [ ] 没有重复的 streamvoice 请求
- [ ] 没有重复的 listWorlds/listSession 请求
- [ ] 请求顺序符合：初始化 -> 加载 -> 开场白 -> 编排 -> 台词 -> 播放

---
