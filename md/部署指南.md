# Toonflow 部署指南

> 最后更新：2026-04-15

## 一、开发环境部署

### 1.1 前置条件

| 软件 | 版本要求 |
|------|----------|
| Node.js | 23.11.1+ (推荐 24.x) |
| Yarn | 1.x |
| Git | 2.x+ |

### 1.2 安装步骤

```bash
# 克隆仓库
git clone https://github.com/HBAI-Ltd/Toonflow-app.git
cd Toonflow-app

# 安装依赖
yarn install
```

### 1.3 启动开发环境

#### 方式一：仅后端 API

```bash
yarn dev
```

- 启动后端服务，端口 60002
- 支持热重载
- **不包含前端页面**，仅 API 调试

#### 方式二：Electron 桌面客户端 (推荐)

```bash
yarn dev:gui
```

- 启动后端 + Electron 窗口
- 自带内置前端页面
- 完整功能体验

#### 方式三：本机环境 GUI

```bash
# PowerShell
yarn local:gui:win:ps

# CMD
yarn local:gui:win:cmd
```

- 使用 local 环境配置
- 适合本地调试

### 1.4 代码质量检查

```bash
# TypeScript 类型检查
yarn lint
```

## 二、生产构建

### 2.1 编译

```bash
yarn build
```

编译输出到 `build/` 目录。

### 2.2 打包桌面应用

#### Windows

```bash
yarn dist:win
```

输出：
- `dist/ToonFlow-{version}-win-x64.exe` (安装程序)
- `dist/ToonFlow-{version}-win-x64-portable.exe` (便携版)

支持架构：x64, arm64

#### macOS

```bash
yarn dist:mac
```

输出：
- `dist/ToonFlow-{version}-mac-x64.dmg`
- `dist/ToonFlow-{version}-mac-arm64.dmg`

#### Linux

```bash
yarn dist:linux
```

输出：
- `dist/ToonFlow-{version}-linux-x64.AppImage`
- `dist/ToonFlow-{version}-linux-x64.deb`

### 2.3 安装包说明

#### Windows NSIS 安装包

- 支持选择安装目录
- 创建桌面快捷方式
- 需要管理员权限

#### macOS DMG

- 拖拽安装到 Applications
- 支持 Intel 和 Apple Silicon

#### Linux AppImage

- 无需安装，直接运行
- 需要执行权限 `chmod +x`

## 三、Docker 部署

### 3.1 前置条件

- Docker 20.10+
- Docker Compose 2.0+

### 3.2 在线部署 (推荐)

从 GitHub/Gitee 自动拉取源码并构建：

```bash
docker-compose -f docker/docker-compose.yml up -d --build
```

#### 构建参数

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `GIT` | 代码仓库源 | github | github / gitee |
| `TAG` | 指定版本标签 | latest | v1.0.6 |
| `BRANCH` | 指定分支 | 默认分支 | main / dev |

#### 版本选择优先级

指定 TAG > 指定 BRANCH > 自动获取最新 tag > 默认分支

#### 示例

```bash
# 使用 Gitee 源 (国内推荐)
GIT=gitee docker-compose -f docker/docker-compose.yml up -d --build

# 指定版本
TAG=v1.0.6 docker-compose -f docker/docker-compose.yml up -d --build

# 指定分支
GIT=gitee BRANCH=dev docker-compose -f docker/docker-compose.yml up -d --build
```

### 3.3 本地构建

使用本地已有源码构建：

```bash
# 先克隆项目
git clone https://github.com/HBAI-Ltd/Toonflow-app.git
cd Toonflow-app

# 本地构建
docker-compose -f docker/docker-compose.local.yml up -d --build
```

### 3.4 端口说明

| 端口 | 用途 | 在线部署映射 | 本地构建映射 |
|------|------|-------------|-------------|
| 80 | Nginx 前端页面 | 随机端口 | 8080:80 |
| 60002 | 后端 API 服务 | 60002:60002 | 60002:60002 |

### 3.5 数据持久化

默认日志目录挂载到宿主机 `./logs`：

```yaml
volumes:
  - ./logs:/var/log
  - ./uploads:/app/uploads      # 持久化上传文件
  - ./data:/app/data            # 持久化数据库
```

### 3.6 常用操作

```bash
# 查看容器状态
docker-compose -f docker/docker-compose.yml ps

# 查看实时日志
docker-compose -f docker/docker-compose.yml logs -f

# 停止服务
docker-compose -f docker/docker-compose.yml down

# 重新构建并启动
docker-compose -f docker/docker-compose.yml up -d --build

# 进入容器调试
docker exec -it toonflow sh
```

## 四、云端服务器部署

### 4.1 服务器环境要求

| 项目 | 要求 |
|------|------|
| 系统 | Ubuntu 20.04+ / CentOS 7+ |
| Node.js | 24.x (推荐) |
| 内存 | 2GB+ |

### 4.2 安装环境

```bash
# 安装 Node.js
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 24

# 安装 Yarn 和 PM2
npm install -g yarn pm2
```

### 4.3 部署项目

```bash
cd /opt

# 从 GitHub 克隆
git clone https://github.com/HBAI-Ltd/Toonflow-app.git
cd Toonflow-app

# 或从 Gitee 克隆 (国内推荐)
git clone https://gitee.com/HBAI-Ltd/Toonflow-app.git
cd Toonflow-app

# 安装依赖并构建
yarn install
yarn build
```

### 4.4 配置 PM2

创建 `pm2.json`：

```json
{
  "name": "toonflow-app",
  "script": "build/app.js",
  "instances": "max",
  "exec_mode": "cluster",
  "env": {
    "NODE_ENV": "prod",
    "PORT": 60002,
    "OSSURL": "http://127.0.0.1:60002/"
  }
}
```

### 4.5 启动服务

```bash
pm2 start pm2.json
pm2 startup
pm2 save
```

### 4.6 常用命令

```bash
pm2 list                    # 查看进程
pm2 logs toonflow-app       # 查看日志
pm2 restart all             # 重启服务
pm2 monit                   # 监控面板
```

## 五、环境变量配置

### 5.1 环境文件

```
env/
├── .env.dev                 # 开发环境
├── .env.local               # 本地环境 (不上传 git)
└── .env.prod                # 生产环境
```

### 5.2 多环境说明

| 环境 | 配置文件 | 用途 | Git |
|------|----------|------|-----|
| dev | env/.env.dev | 快速启动、测试 | 上传 |
| local | env/.env.local | 本机调试 | 不上传 |
| prod | env/.env.prod | 生产发布 | 上传 |

### 5.3 关键环境变量

| 变量 | 说明 | 示例 |
|------|------|------|
| `NODE_ENV` | 运行环境 | dev / local / prod |
| `PORT` | 服务端口 | 60002 |
| `DB_PATH` | 数据库路径 | ./db.sqlite |
| `UPLOAD_DIR` | 上传目录 | ./uploads |
| `OSSURL` | 静态资源访问地址 | http://127.0.0.1:60002/ |

## 六、前端部署

前端为独立的 Vue3 SPA 项目，位于 `Toonflow-game-web/` 仓库。

### 6.1 前端构建

```bash
# 在前端项目目录
cd Toonflow-game-web
yarn install
yarn build
```

### 6.2 集成到后端

将前端构建产物复制到 `scripts/web/` 目录：

```bash
# 复制 dist 目录内容到 scripts/web/
cp -r Toonflow-game-web/dist/* toonflow-game-app/scripts/web/
```

### 6.3 开发模式

开发前端时，直接运行前端项目的开发服务器：

```bash
cd Toonflow-game-web
yarn dev
```

**注意**：开发时不要执行 `yarn build`，以免影响开发效率。

## 九、故障排查

### 9.1 端口占用

```bash
# Windows 查找占用 60002 端口的进程
netstat -ano | findstr :60002

# Linux 查找占用端口
lsof -i :60002
```

### 9.2 数据库问题

```bash
# 备份数据库
cp db.sqlite db.sqlite.backup

# 清除数据库重新开始 (谨慎操作)
rm db.sqlite
```

### 9.3 日志位置

| 部署方式 | 日志位置 |
|----------|----------|
| 开发模式 | 终端输出 |
| 本机运行 | logs/ 目录 |
| Docker | docker-compose logs |
| PM2 | pm2 logs |

### 9.4 常见问题

| 问题 | 解决方案 |
|------|----------|
| 依赖安装失败 | 清除 node_modules 重新安装 |
| 构建失败 | 检查 TypeScript 版本 |
| 端口冲突 | 修改 PORT 环境变量 |
| AI 调用失败 | 检查 API Key 和网络 |

## 十、安全建议

### 10.1 生产环境

- 不要提交真实 API Key 到代码库
- 修改默认管理员密码
- 启用 HTTPS (生产环境)
- 定期备份数据库

### 10.2 敏感文件

以下文件不应提交到版本控制：

- `db.sqlite` (数据库文件)
- `*.bak` (备份文件)
- `env/.env.local` (本地配置)
- `uploads/` (上传文件)

## 十一、版本更新

### 11.1 桌面客户端

1. 下载新版本安装包
2. 运行安装程序
3. 数据自动保留

### 11.2 Docker 更新

```bash
# 重新构建并启动
docker-compose -f docker/docker-compose.yml up -d --build
```

### 11.3 服务器更新

```bash
cd /opt/Toonflow-app
git pull
yarn install
yarn build
pm2 restart all
```

## 十二、回滚方案

### 12.1 桌面客户端

- 卸载当前版本
- 安装旧版本安装包
- 数据不受影响

### 12.2 Docker

```bash
# 指定旧版本
TAG=v1.0.6 docker-compose -f docker/docker-compose.yml up -d --build
```

### 12.3 服务器

```bash
# 回退 Git 提交
git checkout v1.0.6
yarn install
yarn build
pm2 restart all
```
