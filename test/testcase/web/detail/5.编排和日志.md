# 测试用例：编排和日志

## 前置条件
- 已完成登录
- 已进入章节调试或游玩模式

## 测试步骤

### TC-019: 编排业务正确性验证
1. 进入章节调试模式
2. 观察编排请求的触发时机
3. 检查编排请求的内容

**预期结果：**
- 编排请求在正确时机触发（事件激活时）
- 编排请求包含正确的上下文
- 不会重复发送编排请求
- 不会出现 35ms 的异常编排业务

**验收标准：**
- [ ] 编排请求时机正确
- [ ] 请求内容正确
- [ ] 没有重复请求
- [ ] 没有异常编排业务

---

### TC-020: 编排师日志 - 返回内容记录
1. 进入章节调试模式
2. 触发编排请求
3. 查看日志 `[story:orchestrator:runtime]`

**预期结果：**
- 日志中包含编排返回的完整内容
- 返回内容可读

**验收标准：**
- [ ] 日志记录编排返回内容
- [ ] 内容完整可读

---

### TC-021: 编排师日志 - Token 消耗记录
1. 进入章节调试模式
2. 触发编排请求
3. 查看日志 `[story:orchestrator:runtime]`

**预期结果：**
- 日志中包含推理的 token 消耗
- 包含 input_tokens 和 output_tokens

**验收标准：**
- [ ] 日志记录 input_tokens
- [ ] 日志记录 output_tokens
- [ ] 可以计算总消耗

---

### TC-022: 开场白独立性验证
1. 进入章节调试模式
2. 检查第一章节是否有开场白
3. 观察开场白和第一章节内容的关系

**预期结果：**
- 开场白独立于第一章节内容
- 开场白在 introduction 接口返回
- 不会和第一章节内容混淆

**验收标准：**
- [ ] 开场白独立显示
- [ ] 开场白通过 introduction 接口获取
- [ ] 不会重复显示
