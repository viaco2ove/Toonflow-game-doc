# Toonflow 游戏引擎设计

> 最后更新：2026-04-15

## 一、引擎概述

游戏引擎是 Toonflow 项目最复杂的业务模块，位于 `src/modules/game-runtime/`，负责驱动多角色 AI 互动故事的运行时逻辑。

### 设计目标

- 实现类似文字冒险游戏 (Visual Novel) 的互动叙事体验
- 通过 AI 大模型生成动态剧情，而非完全预设脚本
- 在有限上下文和 Token 预算下实现可持续的长程叙事
- 支持章节、任务、触发器等多层次的游戏逻辑

### 核心挑战

> AI 故事的现状就是无法达到理想化的无限输入、无限上下文、无限 Token 和极速返回。
> 所以要进行信息取舍和拆分。

## 三、核心概念

### 3.1 故事世界 (StoryWorld)

故事世界是游戏运行的容器，包含：
- 世界观设定 (背景、规则、风格)
- 角色定义 (用户角色、旁白角色、NPC)
- 章节列表

```typescript
interface StoryWorld {
  id: number;
  name: string;                    // 世界名称
  intro: string;                   // 世界简介
  settings: object;                // 世界设定 JSON
  playerRole: object;              // 用户角色定义
  narratorRole: object;            // 旁白角色定义
}
```

### 3.2 章节 (Chapter)

章节是故事的基本单元，包含：
- 章节内容 (opening / scene / user / fixed / ending)
- 任务列表 (ChapterTask)
- 触发器列表 (ChapterTrigger)
- 进入/完成条件

```typescript
interface StoryChapter {
  id: number;
  chapterKey: string;              // 唯一标识
  title: string;
  content: string;                 // 章节内容
  openingText: string;             // 开场白
  entryCondition: string;          // 进入条件表达式
  completionCondition: string;     // 完成条件表达式
}
```

### 3.3 会话 (Session)

会话是用户与 AI 故事的一次完整互动：

```typescript
interface GameSession {
  sessionId: string;               // UUID 主标识
  worldId: number;
  chapterId: number;               // 当前章节
  status: "active" | "completed" | "failed";
  stateJson: object;               // 完整会话状态
}
```

### 3.4 运行时状态 (Runtime State)

会话状态是整个引擎的核心数据结构，包含：

```typescript
interface RuntimeState {
  player: {
    name: string;
    attributes: Record<string, any>;  // 用户属性 (好感度、属性值等)
  };
  narrator: {
    name: string;
    attributes: Record<string, any>;  // 旁白属性
  };
  npcs: Record<string, {             // NPC 集合
    name: string;
    attributes: Record<string, any>;
  }>;
  vars: Record<string, any>;         // 变量存储
  flags: Record<string, boolean>;    // 标志位
  chapterProgress: ChapterProgressState; // 章节进度
  currentEvent: RuntimeEventDigestState; // 当前事件状态
}
```

```typescript
interface ChapterProgressState {
```

### 4.4 事件类型 (EventKind)

| 类型 | 说明 | 流程阶段 |
|------|------|----------|

| `user` | 用户互动节点 | chapter_content |
| `fixed` | 固定结局检查 | chapter_ending_check |
| `ending` | 章节结局 | chapter_ending_check |

### 4.5 事件状态 (EventStatus)

| 状态 | 说明 |
|------|------|
| `idle` | 未激活 |
| `active` | 进行中 |
| `waiting_input` | 等待用户输入 |
| `completed` | 已完成 |

## 五、引擎模块详解

### 5.1 NarrativeOrchestrator (叙事编排引擎)

**职责**：协调 AI 对话流程，管理回合制叙事

#### 核心流程

```
用户输入
    │
    ▼
┌───────────────────┐
│  resolveSpeakerModeDecision  │
│  决定发言角色       │
└────────┬──────────┘
         │
         ▼
┌───────────────────┐
│  resolveRuleNarrativePlan  │
│  规则驱动的叙事计划 │
└────────┬──────────┘
         │
         ▼
┌───────────────────┐
│  构建 AI Prompt   │
│  (含记忆/上下文)   │
└────────┬──────────┘
         │
         ▼
┌───────────────────┐
│  调用 AI 生成回复  │
│  (支持重试)        │
└────────┬──────────┘
         │
         ▼
┌───────────────────┐
│  parseNarrativeResponse │
│  解析 AI 响应      │
└────────┬──────────┘
         │
         ▼
┌───────────────────┐
│  更新会话状态       │
│  记录消息           │
└───────────────────┘
```

#### 输入输出

```typescript
interface OrchestratorInput {
  userId: number;
  world: StoryWorld;
  chapter: StoryChapter;
  state: JsonRecord;            // 当前运行时状态
  recentMessages: RuntimeMessageInput[]; // 近期消息
  playerMessage?: string;       // 用户输入
  maxRetries?: number;          // AI 重试次数
  allowControlHints?: boolean;  // 是否允许控制提示
  allowStateDelta?: boolean;    // 是否允许状态变更
}

interface NarrativePlanResult {
  role: string;                 // 发言角色
  roleType: string;             // 角色类型
  motive: string;               // 发言动机
  memoryHints: string[];        // 记忆提示
  triggerMemoryAgent: boolean;  // 是否触发记忆代理
}
```

#### 叙事回合 (Turn)

每次叙事循环称为一个 Turn，包含：
1. 接收用户输入
2. 决定发言角色
3. 调用 AI 生成回复
4. 更新状态
5. 检查触发器

### 5.2 TriggerEngine (触发器引擎)

**职责**：基于条件表达式的事件触发系统

#### 触发器结构

```typescript
interface ChapterTrigger {
  id: number;
  chapterId: number;
  name: string;                  // 触发器名称
  triggerEvent: string;          // 触发事件类型
  conditionExpr: string;         // 条件表达式
  actionExpr: string;            // 动作表达式
  enabled: number;               // 是否启用
}
```

#### 触发事件类型

| 事件类型 | 触发时机 |
|----------|----------|
| `message_sent` | 消息发送后 |
| `user_input` | 用户输入后 |
| `ai_response` | AI 回复后 |
| `turn_complete` | 回合结束时 |
| `chapter_enter` | 进入章节时 |
| `chapter_exit` | 离开章节时 |

#### 条件表达式

使用 `evaluateCondition()` 进行表达式求值：

```
// 示例条件表达式
"player.attributes.affection >= 50"
"flags.quest_completed == true"
"vars.key_item_count > 0"
```

#### 动作表达式

动作执行通过 `applyAttributeChanges()` 实现：

```json
{
  "attrChanges": [
    {
      "entityType": "player",
      "field": "affection",
      "value": 60
    },
    {
      "entityType": "flags",
      "field": "met_villager",
      "value": true
    }
  ]
}
```

#### 执行流程

```
事件发生
    │
    ▼
遍历章节触发器
    │
    ▼
检查 enabled == 1
    │
    ▼
检查 triggerEvent 匹配
    │
    ▼
求值 conditionExpr
    │
    ▼ (条件满足)
执行 actionExpr
    │
    ▼
应用状态变更 (AppliedDelta)
    │
    ▼
返回触发结果 (TriggerHit)
```

### 5.3 ChapterProgressEngine (章节进度引擎)

**职责**：管理章节进度状态机的转换

#### 核心函数

| 函数 | 说明 |
|------|------|
| `readChapterProgressState` | 读取当前章节进度 |
| `setChapterProgressState` | 设置章节进度 |
| `advanceChapterProgressAfterNarrative` | 叙事完成后推进进度 |
| `applyAiEventProgressResolution` | 应用 AI 事件进度决议 |
| `syncChapterProgressWithRuntime` | 同步运行时状态 |

#### 进度推进流程

```

    │
    ▼

    │ (AI 评估事件进度)
    ▼
applyAiEventProgressResolution
    │
    ▼

    │
    ▼

    │
    ▼
syncRuntimeCurrentEventFromChapterProgress
```

### 5.4 TaskProgressEngine (任务进度引擎)

**职责**：跟踪章节内任务的完成状态

#### 任务状态

| 状态 | 说明 |
|------|------|
| `pending` | 待开始 |
| `in_progress` | 进行中 |
| `completed` | 已完成 |
| `failed` | 失败 |

#### 任务结构

```typescript
interface ChapterTask {
  id: number;
  chapterId: number;
  title: string;
  taskType: string;
  status: string;
  successCondition: string;    // 成功条件
  failCondition: string;       // 失败条件
  rewardAction: string;        // 奖励动作
  goalType: string;
  parentTaskId: number;        // 父任务 (支持子任务)
}
```

#### 任务推进

```
消息输入
    │
    ▼
检查任务 successCondition
    │
    ▼ (满足)
更新任务状态 → completed
    │
    ▼
执行 rewardAction
    │
    ▼
检查父任务是否可完成
```

### 5.5 SpeakerRouteEngine (发言路由引擎)

**职责**：决定每个叙事回合由哪个角色发言

#### 发言模式

| 模式 | 说明 |
|------|------|
| `player_turn` | 用户回合 (等待用户输入) |
| `narrator_turn` | 旁白回合 (AI 生成叙事) |
| `npc_turn` | NPC 回合 (特定 NPC 发言) |

#### 决策流程

```
当前状态 + 用户输入
    │
    ▼
resolveSpeakerModeDecision
    │
    ├── 检查是否在用户节点
    │   └── 是 → player_turn
    │
    ├── 检查是否有 NPC 需要发言
    │   └── 是 → npc_turn
    │
    └── 默认 → narrator_turn
```

### 6.4 SessionMemoryWorker (后台记忆 Worker)

**职责**：异步处理会话记忆，优化上下文

#### 工作机制

```
后台定时执行
    │
    ▼
检查活跃会话
    │
    ▼
提取记忆事实
    │
    ├── 重要事件摘要
    ├── 角色关系变化
    └── 关键决策记录
    │
    ▼
更新 memorySummary / memoryFacts
    │
    ▼
清理过期记忆
```

#### 设计要点

> 由于上下文限制，不能保存所有历史消息。
> 通过异步 Worker 提取关键记忆，保持上下文精简但信息完整。

## 十、性能优化

### 10.1 上下文窗口管理

| 策略 | 说明 |
|------|------|
| 记忆摘要 | 将历史消息压缩为摘要 |
| 事实提取 | 提取关键事实而非保存全文 |
| 窗口滑动 | 只保留最近 N 条完整消息 |
| 按需加载 | 需要时从快照恢复历史 |

### 10.2 异步处理

```
SessionMemoryWorker (后台定时)
    │
    ├── 提取记忆摘要
    ├── 更新事实列表
    └── 清理过期数据

主线程 (叙事循环)
    │
    ├── 读取最新记忆
    ├── 构建 Prompt
    └── 调用 AI
```

### 10.3 状态快照优化

| 场景 | 策略 |
|------|------|
| 频繁保存 | 只保存增量变更 (entityStateDelta) |
| 按需恢复 | 从最近快照 + 增量变更重建状态 |
| 快照清理 | 保留最近 N 个快照 |
