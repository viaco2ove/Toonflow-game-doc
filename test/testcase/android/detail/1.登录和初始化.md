# 登录和初始化测试

## TC-A01: 登录测试

### 测试目的
验证安卓端能否正确登录并获取 token

### 测试步骤
1. 调用 `POST /other/login`
2. 传递参数: `{ username: "admin", password: "admin123" }`
3. 验证返回的 token 格式

### 预期结果
- ✅ 返回 code: 200
- ✅ 返回 data.token 字段
- ✅ token 格式为 "Bearer xxx..."

### 安卓端实现
```kotlin
// GameApi.kt
@POST("other/login")
suspend fun login(@Body payload: JsonObject): ApiEnvelope<JsonObject>

// GameRepository.kt
suspend fun login(username: String, password: String): JsonObject {
  val payload = JsonObject().apply {
    addProperty("username", username)
    addProperty("password", password)
  }
  return unwrapEnvelope("other/login", api().login(payload))
}
```

### 测试目的

### 测试步骤
1. 使用 token 调用 `POST /game/listWorlds`

## TC-A03: 初始化调试模式

### 测试目的
验证安卓端能否正确初始化调试模式（不包含开场白）

### 测试步骤
1. 调用 `POST /game/initDebug`
2. 传递参数: `{ worldId: X, chapterId: Y }`
3. 验证返回的初始状态

### 预期结果
- ✅ 返回 code: 200
- ✅ 返回 data.state 包含 debugRuntimeKey
- ✅ **不再返回 opening 和 firstChapter 字段**（接口分离）
- ✅ state.chapterProgress 包含初始事件信息

### 安卓端数据模型
```kotlin
data class InitDebugResult(
  val chapterId: Long?,
  val chapterTitle: String,
  val state: JsonElement?,
  val endDialog: String?,
  val endDialogDetail: String?,
  // 注意：不再包含 opening 和 firstChapter 字段
)
```

### 关键验证点
- `state.chapterProgress.eventIndex` 应为初始值（1 或 null）
- `state.chapterProgress.eventKind` 应为 "opening" 或 "scene"
- `state.turnState.canPlayerSpeak` 应正确反映当前轮次
- **必须先调用 initDebug，再调用 introduction 获取开场白**

## TC-A04: 获取开场白

### 测试目的
验证安卓端能否通过 introduction 接口获取开场白

### 测试步骤
1. 调用 `POST /game/introduction`
2. 传递参数: `{ worldId: X, chapterId: Y, state: initState }`
3. 验证返回的 plan 字段

### 预期结果
- ✅ 返回 code: 200
- ✅ 返回 data.plan 字段
- ✅ plan.eventType 为 "on_opening"
- ✅ plan.presetContent 包含开场白内容

### 安卓端数据模型
```kotlin
data class IntroductionResult(
  val chapterId: Long?,
  val chapterTitle: String,
  val state: JsonElement?,
  val currentEventDigest: RuntimeEventDigestItem?,
  val eventDigestWindow: List<RuntimeEventDigestItem>,
  val eventDigestWindowText: String,
  val plan: PlanItem?,  // 开场白计划
)

data class PlanItem(
  val role: String,
  val roleType: String,
  val motive: String?,
  val awaitUser: Boolean,
  val eventType: String,
  val presetContent: String?,  // 开场白内容
  val planSource: String?,
)
```

### 关键验证点
- plan.eventType 应为 "on_opening"
- plan.presetContent 应有内容
- 前端需要将 plan 转换为消息显示

## 测试脚本参考

```javascript
// TC-A01
const loginRes = await fetch('http://127.0.0.1:60002/other/login', {

console.log('Worlds:', worldsData.data);

    'Authorization': loginData.data.token
  },
  body: JSON.stringify({ worldId: 1, chapterId: 1 })
});
const initData = await initRes.json();

// TC-A04: 获取开场白
const introRes = await fetch('http://127.0.0.1:60002/game/introduction', {
```
