# 游戏流程测试

## TC-A04: 获取开场白

### 测试目的
验证安卓端能否正确获取章节开场白

### 测试步骤
1. 调用 `POST /game/debugStep`
2. 不传 playerContent，只传 worldId、chapterId、state、messages
3. 验证返回的开场白消息

### 预期结果
- ✅ 返回 code: 200
- ✅ messages 包含开场白消息
- ✅ 开场白消息有 `eventType: "on_opening"` 标记
- ✅ state.turnState.canPlayerSpeak 正确设置

### 安卓端数据模型
```kotlin
data class MessageItem(
  val id: Long,
  val role: String = "",
  val roleType: String = "",
  val eventType: String = "",
  val content: String = "",
  val createTime: Long = 0L,
  val meta: JsonElement? = null,
  val revisitData: JsonElement? = null,
)
```

## TC-A05: 用户输入推进剧情

### 测试目的
验证安卓端能否正确处理用户输入并推进剧情

### 测试步骤
1. 检查 state.turnState.canPlayerSpeak 是否为 true
2. 调用 `POST /game/debugStep`
3. 传递 playerContent: "用户输入内容"
4. 验证返回的剧情消息

### 预期结果
- ✅ 返回 code: 200
- ✅ messages 包含用户消息和旁白回复
- ✅ state.chapterProgress.eventIndex 递增
- ✅ 用户消息有 `eventType: "on_message"`

### 关键验证点
- 用户消息的 `role` 应与 `playerRole.name` 一致
- 旁白回复的 `roleType` 应为 "narrator"
- `state.chapterProgress` 应正确更新

## TC-A06: 旁白继续发言

### 测试目的
验证安卓端能否正确处理旁白连续发言

### 测试步骤
1. 不传 playerContent
2. 调用 `POST /game/debugStep`
3. 验证旁白是否继续发言

### 预期结果
- ✅ 返回 code: 200
- ✅ messages 包含旁白消息
- ✅ state.turnState.canPlayerSpeak 可能变为 true
- ✅ 消息的 `eventType` 为 "on_orchestrated_reply"

### 安卓端处理逻辑
```kotlin
// 判断是否可以用户发言
val canSpeak = state.asJsonObject
  .get("turnState")?.asJsonObject
  ?.get("canPlayerSpeak")?.asBoolean ?: false

if (!canSpeak) {
  // 不传 playerContent，让旁白继续发言
  val stepRes = api().debugStep(buildStepPayload(
    worldId = worldId,
    chapterId = chapterId,
    state = state,
    messages = messages,
    playerContent = null
  ))
}
```

## TC-A07: 章节结束判定

### 测试目的
验证安卓端能否正确处理章节结束

### 测试步骤
1. 触发章节完成条件
2. 验证 endDialog 和 endDialogDetail 字段
3. 验证是否进入下一章节

### 预期结果
- ✅ 返回 endDialog 字段（"已成功" 或 "已失败"）
- ✅ 返回 endDialogDetail 包含详细信息
- ✅ 如果有下一章，chapterId 更新
- ✅ 如果没有下一章，进入自由剧情

### 安卓端数据模型
```kotlin
data class DebugStepResult(
  // ...
  val endDialog: String?,
  val endDialogDetail: String?,
  // ...
)
```

### 结束类型
1. **已成功**: 章节完成，进入下一章或自由剧情
2. **已失败**: 章节失败，游戏结束
3. **进入自由剧情**: 所有章节完成，进入沙盒模式

## 安卓端状态管理要点

### 消息累积
安卓端需要维护一个消息列表：
```kotlin
private val messages = mutableListOf<MessageItem>()

fun addMessage(message: MessageItem) {
  messages.add(message)
  // 更新 UI
}
```

### 状态传递
每次调用 debugStep 都需要传递最新的 state：
```kotlin
private var currentState: JsonElement? = null

suspend fun performStep(playerContent: String?) {
  val res = repository.debugStep(
    worldId = worldId,
    chapterId = chapterId,
    state = currentState,
    messages = messages,
    playerContent = playerContent
  )
  currentState = res.state
  res.messages.forEach { messages.add(it) }
}
```
