# Toonflow 数据库设计

> 最后更新：2026-04-15

## 一、数据库概览

- **数据库类型**：SQLite3 (嵌入式)
- **ORM**：Knex.js (查询构建器)
- **驱动**：better-sqlite3 (同步高性能)
- **数据库文件**：`db.sqlite` (项目根目录)
- **类型定义**：`src/types/database.d.ts` (自动生成)

## 三、核心表详细设计

### 3.1 t_user (用户表)

```sql
CREATE TABLE t_user (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT,                    -- 用户名
    password TEXT,                -- 密码 (加密存储)
    avatarPath TEXT,              -- 头像路径
    avatarBgPath TEXT             -- 头像背景路径
);
```

**设计要点**：
- 默认管理员用户：admin / admin123
- 每个用户独立的 tokenKey 用于 JWT 签名

### 3.2 t_project (项目表)

```sql
CREATE TABLE t_project (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT,                    -- 项目名称
    intro TEXT,                   -- 项目简介
    type TEXT,                    -- 项目类型
    artStyle TEXT,                -- 美术风格
    videoRatio TEXT,              -- 视频比例 (16:9, 9:16)
    userId INTEGER,               -- 所属用户
    createTime INTEGER            -- 创建时间戳
);
```

**设计要点**：
- 通过 userId 实现用户数据隔离
- artStyle 影响 AI 图像生成的风格

### 3.3 t_novel (小说章节表)

```sql
CREATE TABLE t_novel (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    projectId INTEGER,            -- 所属项目
    chapter TEXT,                 -- 章节标题
    chapterIndex INTEGER,         -- 章节序号
    chapterData TEXT,             -- 章节内容 JSON
    reel TEXT,                    -- 卷/篇
    createTime INTEGER            -- 创建时间戳
);
```

**设计要点**：
- chapterData 存储结构化章节数据
- 支持多卷多章节组织

### 3.4 t_outline (剧情大纲表)

```sql
CREATE TABLE t_outline (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    projectId INTEGER,            -- 所属项目
    episode INTEGER,              -- 集数/章节
    data TEXT                     -- 大纲内容 JSON
);
```

**设计要点**：
- data 字段存储 AI 生成的完整大纲结构
- 一个项目可有多个大纲版本

### 3.5 t_script / t_scriptSegment (剧本及片段)

```sql
CREATE TABLE t_script (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    projectId INTEGER,
    name TEXT,                    -- 剧本名称
    content TEXT,                 -- 完整剧本内容
    outlineId INTEGER             -- 关联大纲
);

CREATE TABLE t_scriptSegment (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    projectId INTEGER,
    scriptId INTEGER,             -- 所属剧本
    title TEXT,                   -- 片段标题
    content TEXT,                 -- 片段内容
    summary TEXT,                 -- 片段摘要
    sort INTEGER,                 -- 排序序号
    startAnchor TEXT,             -- 开始锚点
    endAnchor TEXT,               -- 结束锚点
    createTime INTEGER,
    updateTime INTEGER
);
```

**设计要点**：
- 一个剧本包含多个片段 (Segment)
- 锚点用于精确定位片段在剧本中的位置

### 3.6 t_assets (素材表)

```sql
CREATE TABLE t_assets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    projectId INTEGER,
    scriptId INTEGER,
    segmentId INTEGER,
    name TEXT,                    -- 素材名称
    intro TEXT,                   -- 素材简介
    type TEXT,                    -- 类型 (image, audio, etc)
    state TEXT,                   -- 状态
    prompt TEXT,                  -- 生成提示词
    videoPrompt TEXT,             -- 视频提示词
    filePath TEXT,                -- 文件路径
    duration TEXT,                -- 时长
    episode TEXT,                 -- 集数
    shotIndex INTEGER,            -- 镜头序号
    voiceConfig TEXT,             -- 语音配置
    remark TEXT                   -- 备注
);
```

**设计要点**：
- 统一管理所有类型的素材
- prompt/videoPrompt 用于追溯生成参数

### 3.7 t_video / t_videoConfig (视频及配置)

```sql
CREATE TABLE t_videoConfig (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    projectId INTEGER,
    scriptId INTEGER,
    prompt TEXT,                  -- 视频提示词
    images TEXT,                  -- 参考图片 JSON 数组
    duration INTEGER,             -- 时长 (秒)
    manufacturer TEXT,            -- 厂商
    mode TEXT,                    -- 模式
    resolution TEXT,              -- 分辨率
    state INTEGER,                -- 状态
    selectedResultId INTEGER,     -- 选中的结果
    startFrame TEXT,              -- 首帧
    endFrame TEXT,                -- 尾帧
    audioEnabled INTEGER,         -- 是否启用音频
    audioPath TEXT,               -- 音频路径
    dialogue TEXT,                -- 对白
    voiceConfigId INTEGER,        -- 语音配置 ID
    voicePresetId TEXT,           -- 语音预设 ID
    sort INTEGER,
    createTime INTEGER,
    updateTime INTEGER
);

CREATE TABLE t_video (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    configId INTEGER,             -- 关联配置
    aiConfigId INTEGER,           -- AI 配置 ID
    filePath TEXT,                -- 视频文件路径
    firstFrame TEXT,              -- 首帧图片
    state INTEGER,                -- 状态 (0:待处理, 1:处理中, 2:完成, 3:失败)
    prompt TEXT,                  -- 使用的提示词
    model TEXT,                   -- 使用模型
    providerManufacturer TEXT,    -- 提供商厂商
    providerTaskId TEXT,          -- 提供商任务 ID (用于轮询)
    providerQueryUrl TEXT,        -- 查询 URL
    errorReason TEXT,             -- 失败原因
    storyboardImgs TEXT,          -- 分镜图片 JSON
    resolution TEXT,
    time INTEGER                  -- 时长
);
```

**设计要点**：
- 配置与结果分离，支持多次生成选择最佳
- providerTaskId 用于异步任务轮询
- startFrame/endFrame 支持首尾帧控制视频生成

### 3.8 t_storyWorld (故事世界表)

```sql
CREATE TABLE t_storyWorld (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    projectId INTEGER,
    name TEXT,                    -- 世界名称
    intro TEXT,                   -- 世界简介
    coverPath TEXT,               -- 封面路径
    settings TEXT,                -- 世界设定 JSON
    playerRole TEXT,              -- 用户角色 JSON
    narratorRole TEXT,            -- 旁白角色 JSON
    publishStatus TEXT,           -- 发布状态
    createTime INTEGER,
    updateTime INTEGER
);
```

**设计要点**：
- settings 存储世界规则和背景设定
- playerRole/narratorRole 定义角色行为模式
- 支持发布状态用于分享

### 3.9 t_storyChapter (故事章节表)

```sql
CREATE TABLE t_storyChapter (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    worldId INTEGER,              -- 所属世界
    chapterKey TEXT,              -- 章节唯一键
    title TEXT,                   -- 章节标题
    content TEXT,                 -- 章节内容
    openingText TEXT,             -- 开场白
    openingRole TEXT,             -- 开场角色
    backgroundPath TEXT,          -- 背景图
    bgmPath TEXT,                 -- 背景音乐
    bgmAutoPlay INTEGER,          -- BGM 自动播放
    runtimeOutline TEXT,          -- 运行时大纲 JSON
    entryCondition TEXT,          -- 进入条件
    completionCondition TEXT,     -- 完成条件
    showCompletionCondition INTEGER, -- 显示完成条件
    sort INTEGER,                 -- 排序
    status TEXT,                  -- 状态
    createTime INTEGER,
    updateTime INTEGER
);
```

**设计要点**：
- entryCondition/completionCondition 表达式用于触发器判定
- runtimeOutline 存储运行时的动态大纲

### 3.10 t_gameSession (游戏会话表)

```sql
CREATE TABLE t_gameSession (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    userId INTEGER,
    projectId INTEGER,
    worldId INTEGER,
    sessionId TEXT,               -- 会话唯一 ID (UUID)
    chapterId INTEGER,            -- 当前章节
    title TEXT,                   -- 会话标题
    status TEXT,                  -- 状态 (active, completed, failed)
    stateJson TEXT,               -- 会话状态 JSON
    contentVersion TEXT,          -- 内容版本
    createTime INTEGER,
    updateTime INTEGER
);
```

**设计要点**：
- sessionId 作为会话的主标识
- stateJson 存储完整会话状态快照
- contentVersion 用于状态迁移兼容

### 3.11 t_sessionMessage (会话消息表)

```sql
CREATE TABLE t_sessionMessage (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sessionId TEXT,               -- 会话 ID
    role TEXT,                    -- 角色 (user, assistant, system, narrator)
    roleType TEXT,                -- 角色类型
    content TEXT,                 -- 消息内容
    eventType TEXT,               -- 事件类型
    meta TEXT,                    -- 元数据 JSON
    revisitData TEXT,             -- 回溯数据 JSON
    createTime INTEGER
);
```

**设计要点**：
- eventType 区分普通对话、事件触发、系统通知等
- revisitData 支持消息回溯修改
- meta 存储额外信息如语音配置、图片引用

### 3.12 t_sessionStateSnapshot (会话状态快照表)

```sql
CREATE TABLE t_sessionStateSnapshot (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sessionId TEXT,
    round INTEGER,                -- 回合数
    stateJson TEXT,               -- 状态 JSON
    reason TEXT,                  -- 快照原因
    createTime INTEGER
);
```

**设计要点**：
- 定期创建快照，支持回滚
- round 表示对话轮次
- reason 记录快照触发原因 (定时/事件/手动)

### 3.13 t_entityStateDelta (实体状态变更表)

```sql
CREATE TABLE t_entityStateDelta (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sessionId TEXT,
    entityId TEXT,                -- 实体 ID
    entityType TEXT,              -- 实体类型 (role, world, chapter)
    field TEXT,                   -- 变更字段
    oldValue TEXT,                -- 旧值
    newValue TEXT,                -- 新值
    eventId TEXT,                 -- 关联事件 ID
    source TEXT,                  -- 变更来源
    createTime INTEGER
);
```

**设计要点**：
- 追踪所有实体状态的变更历史
- 支持状态审计和回溯
- eventId 关联到触发变更的事件

### 3.14 t_chapterTask (章节任务表)

```sql
CREATE TABLE t_chapterTask (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    chapterId INTEGER,
    title TEXT,                   -- 任务标题
    taskType TEXT,                -- 任务类型
    status TEXT,                  -- 状态 (pending, in_progress, completed, failed)
    successCondition TEXT,        -- 成功条件表达式
    failCondition TEXT,           -- 失败条件表达式
    rewardAction TEXT,            -- 奖励动作表达式
    goalType TEXT,                -- 目标类型
    parentTaskId INTEGER,         -- 父任务 ID (子任务)
    sort INTEGER,
    createTime INTEGER,
    updateTime INTEGER
);
```

**设计要点**：
- 支持任务嵌套 (parentTaskId)
- 条件表达式由 TriggerEngine 求值

### 3.15 t_chapterTrigger (章节触发器表)

```sql
CREATE TABLE t_chapterTrigger (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    chapterId INTEGER,
    name TEXT,                    -- 触发器名称
    triggerEvent TEXT,            -- 触发事件
    conditionExpr TEXT,           -- 条件表达式
    actionExpr TEXT,              -- 动作表达式
    enabled INTEGER,              -- 是否启用 (1/0)
    sort INTEGER,
    createTime INTEGER,
    updateTime INTEGER
);
```

**设计要点**：
- 当 triggerEvent 发生时检查 conditionExpr
- 条件满足则执行 actionExpr
- 支持启用/禁用开关

### 3.16 t_config (AI 模型配置表)

```sql
CREATE TABLE t_config (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    userId INTEGER,
    manufacturer TEXT,            -- 厂商标识

    apiKey TEXT,                  -- API 密钥
    type TEXT,                    -- 类型
    currency TEXT,                -- 货币单位
    inputPricePer1M INTEGER,      -- 输入价格 (每百万 token)
    outputPricePer1M INTEGER,     -- 输出价格
    cacheReadPricePer1M INTEGER,  -- 缓存读取价格
    reasoningEffort TEXT,         -- 推理强度
    createTime INTEGER
);
```

**设计要点**：
- 支持多厂商多模型配置
- 价格信息用于成本统计
