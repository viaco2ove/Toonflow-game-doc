# 测试用例：回溯功能

## 前置条件
- 已完成登录
- 已进入章节调试或游玩模式
- 已有至少 3 句台词播放完成

## 测试步骤

### TC-008: 章节调试模式回溯 - 内存缓存
1. 进入章节调试模式
2. 播放至少 5 句台词
3. 点击"回溯"按钮
4. 选择第 3 句台词进行回溯

**预期结果：**
- 显示回溯按钮，可点击
- 点击后显示可回溯的台词列表
- 选择后成功回溯到该台词
- 动态参数（如用户输入的名称）正确恢复

**验收标准：**
- [ ] 回溯按钮可用
- [ ] 显示所有已播放台词
- [ ] 回溯成功，显示对应台词
- [ ] 动态参数正确恢复
- [ ] 优先从内存缓存恢复

---

### TC-009: 章节调试模式回溯 - 临时文件缓存
1. 进入章节调试模式
2. 播放至少 5 句台词
3. 刷新页面（模拟内存丢失）
4. 点击"回溯"按钮
5. 选择第 3 句台词进行回溯

**预期结果：**
- 刷新后仍能回溯
- 从临时文件恢复台词缓存
- 动态参数正确恢复

**验收标准：**
- [ ] 刷新后回溯按钮仍可用
- [ ] 从临时文件恢复成功
- [ ] 动态参数正确恢复
- [ ] 临时文件在退出调试时销毁

---

### TC-010: 章节调试模式回溯 - 无缓存提示
1. 进入章节调试模式
2. 只播放 1 句台词
3. 清除浏览器缓存
4. 点击"回溯"按钮
5. 尝试回溯到不存在的台词

**预期结果：**
- 提示"缺少台词记忆"
- 不会崩溃或卡死

**验收标准：**
- [ ] 友好提示缺少台词记忆
- [ ] 应用正常运行

---

### TC-011: 游玩模式回溯 - 持久化恢复
1. 进入游玩模式
2. 播放至少 5 句台词
3. 退出故事
4. 重新进入同一个故事
5. 点击"回溯"按钮
6. 选择第 3 句台词进行回溯

**预期结果：**
- 重新进入后仍能回溯到之前的台词
- 从持久化字段恢复台词缓存
- 动态参数正确恢复

**验收标准：**
- [ ] 重新进入后回溯可用
- [ ] 从持久化数据恢复成功
- [ ] 动态参数正确恢复

---

### TC-012: 回溯后继续播放验证
1. 进入章节调试模式
2. 播放 5 句台词
3. 回溯到第 3 句
4. 继续播放

**预期结果：**
- 从第 3 句开始继续播放
- 后续台词正确生成
- 事件状态正确更新

**验收标准：**
- [ ] 从回溯点继续播放
- [ ] 台词正确生成
- [ ] 事件状态正确
